# Power-mode 仕様・テスト記述スタイルガイド（テンプレート）

本書は電源モード関連の要件を **説明文（Description）／前提（Given）／期待結果（Expectation）** に落とす際の共通ルールです。自動生成・レビュー（IR および `validate_spec.py`）と併用してください。

---

## 1. 用語集テンプレート（JP／EN）

以下の表を複製し、プロジェクトごとに **正式名称／略称／禁止語** を埋めてください。

| JP（ユーザー向け表記） | EN（開発・ログ向け） | 定義・境界 | 近接語との差 | メモ |
| ------------------------ | -------------------- | --------- | ----------- | ----- |
| （例）アクセサリ電源 | ACC power rail | KL15 が OFF であっても許容される定格以下の電源域 | RUN（走行準備）とはイグニション状態が異なる | BCM 依存 |
| （例）スリープ移行 | Transition to sleep | すべての妨害条件が解除され、タイマまたはイベントで低消費電力状態へ遷移 | 「スタンバイ」と混同しない | NVM との境界を明確化 |
| （例）妨害要因 | Sleep inhibit factor | スリープを保留または禁止する入力・診断・ネットワーク要因の総称 | 単なる「警告灯」と混同しない | 一覧はインデックス側で IR へ |

※ 実データの貼り付けではなく、このテンプレートにプロジェクト固有語を追加してください。

## 2. トーンと文体

| 項目 | 推奨 |
| ------ | ------ |
| 文体 | 「〜すること」「〜とする」を基本とした **調書調（だ・である調）**。顧客向け資料と揃える場合のみ **です・ます調** に統一。 |
| 主語 | 可能な限り **ECU／機能ブロック名** を明示し、曖昧な「システム」は避ける。 |
| 数値・単位 | SI 準拠。電圧 `[V]`、時間 `[ms]`／`[s]` は列挙し、単位省略禁止。 |
| 条件結合 | `かつ`/`または` を自然文で書き、必要なら括弧 `( )` で評価順を明示。 |
| 否定的要件 | 「〜しない」だけでなく **許容される状態集合** を補う（監視・テスト可能にするため）。 |

## 3. 近傍ケースとの一意性（Uniqueness vs neighbors）

要件が似た条文を並べるほどレビューコストが増えます。**次を必ず明文化**してください。

1. **適用車種／ECU バリアント**：`applies_to` と同義の粒度で Given の先頭に置く。
2. **状態遷移のきっかけ**：信号エッジ（立ち上がり／下がり／レベル）を区別する。
3. **タイミング**：デバウンス、ホールド、タイムアウトを期待結果側に明示（IR の `timing` と対応）。
4. **例外パス**：診断、工場モード、テレマティックス起動など隣接仕様との差を 1 行で書き分ける。

---

## 4. 敬体（です・ます）を使う場合の注意（任意）

顧客向けドキュメントとテストトリプレットを **文体で揃える** 場合のみ敬体へ統一し、開発者向け内部テストだけである場合は調書調のままで構いません。**同一セクション・同一テストスイート内で文体を混在させない** ことを優先してください。

## 5. ゴールド例：**合成サンプル**（実データではありません）

以下は記述フォーマットの参考用です。**架空の名前・値**であり、製品要件を示すものではありません。

### トリプレット A（ACC と電圧妨害）

**Description**: ゲートウェイ電源マネージャは、補助イグニションが許容状態かつ電池電圧が最低動作電圧以上でデバウンスされた場合に、アクセサリ電源レールを許可する。

**Given**: `IGN_AUX == true`、`BATT_VOLTAGE >= 9.0 V` を 50 ms デバウンス後に成立している。適用：`Gateway ECU / power-manager v2`。

**Expectation**: アクセサリ電源が許可され、ログに `ACC_RAIL_ALLOWED` が一度だけ記録される（チタリングしない）。

---

### トリプレット B（エンジン停止後の RUN 終了）

**Description**: BCM の付帯電源ウィンドウは、エンジンがオフとなった後も一定条件では維持されるが、運転席ドアが開いた時点で閉じる。

**Given**: `ENGINE_STATE == OFF` かつ `DRIVER_DOOR` が OPEN に遷移した直後。適用：`BCM retention policy`。

**Expectation**: 付帯電源ウィンドウがクローズされ、許可済みのアクセサリ負荷は順次シャットダウンシーケンスへ入る。

---

### トリプレット C（ネットワーク・スリープ妨害）

**Description**: CAN 協調において、アウェイク要求ビットマスクが零でない間は協調スリープへ移行しない。

**Given**: `AWAKE_CLIENT_BITMAP != 0`。適用：`Network coordinator`。

**Expectation**: スリープ候補ステートが抑止され、妨害クリア後にのみスリープ交渉シーケンスが再開される。

## 6. IR・ツールチェーンとの対応

| トリプレット要素 | IR フィールド／備考 |
| ---------------- | --------------------- |
| Description | `title` と自然言語要約としてレビュアが参照 |
| Given | `predicates`（複数要素は AND 既定、OR は文中で明示または IR を分割） |
| Expectation | 自動テストの Assert 側；信号名は Given と語彙を合わせる |
| 適用表記 | `applies_to` |
| 出典 | `source_evidence[]` |
